# Self-Improvement — Capture Learnings

After any correction, mistake, or unexpected behavior in this session, create a file in `.claude/learnings/` documenting:

1. **What happened** — describe the error or correction
2. **Why it was wrong** — root cause or misunderstanding
3. **The fix applied** — what was done to resolve it
4. **Rule to follow** — the principle to prevent recurrence

## File Format
- Filename: `YYYY-MM-DD-{slug}.md` (e.g., `2026-03-16-week1-csv-isolation.md`)
- One learning per file
- Keep it concise — 5-15 lines max

## Promotion
Promote critical learnings to `.claude/rules/` if they should be enforced every session (not just referenced when relevant). A learning becomes a rule when violating it would cause data loss, incorrect output, or broken systems.
